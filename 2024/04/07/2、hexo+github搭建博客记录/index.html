hexo+github 博客搭建及常用指令记录

Created2024-04-07|Updated2024-04-12
|Post Views:

前言

应该大一就开始做的。。很喜欢系统地组织、整理自己的学习轨迹，编写技术博客就是一个很好的方式，但是一直担心自己写的笔记不够逻辑通畅而不敢开始。总要踏出第一步吧！那么就从搭建博客开始~

这个博客是基于Github pages+hexo框架搭建的，自己购买服务器和申请域名这些操作都不是必要的，主打的就是一个快捷快速快上手。由于我急于找一个平台扔记录自己的学习笔记，因此选择了这个（自认为）最简单的模型（？）如果有兴趣的话可以试试更完整的搭建自己博客的流程~

*搭建的过程参考了很多他人的记录，感谢华哥在线指导~

环境搭建

1、git

  • 进入git官方下载地址下载安装包，进行安装(点击下一步)
    由于不喜欢往C盘放东西，我放在了D盘里
  • 安装好后，打开Git Bash，输入git -v，出现下图说明安装成功
    ![01](2、nodejs-hexo-github搭建博客记录/git-version.png)

2、node.js

  • 进入nodejs官网下载地址下载对应安装包，进行安装(点击下一步)
    • 同样由于不喜欢放C盘，我又放在了D盘自己新建的node-js文件夹里
    • 博主之后把blog的源文件地址也放在了这个nodejs的文件夹里，感觉怪怪的，有看到其他人是专门建了一个Blog的大文件夹，然后把nodejs的安装地址放在了这个大文件夹里，感觉那样的组织方式更舒服一点？
  • 安装好后的nodejs文件夹里可能只有node_modules文件夹，没有node_global和node_cache，需要手动新建一下，最后文件夹里除了零碎的文件，一共有三个文件夹
  • 配置环境变量
    • 新建用户变量：找到并双击Path→此时会发现自动添加了一个末尾为npm的环境变量，删除→新建→node_global文件夹的地址
    • 新建系统变量1：这个指向的是node_global文件夹内的node_modules文件夹，建好后可以去node_global文件夹里检查一下有没有自动新建这个modules文件夹，如果没有的话就手动建一个
    • 新建系统变量2：找到并双击Path→新建→如图所示
      博主在这一步卡了很久，看了很多教程，最终成功时一共新建了三条path，如果嫌多的话可以做控制变量实验看看具体是哪一条或哪几条起了作用:P
- 完成后在终端里输入node -v，出现下图说明安装成功

3、Hexo

  • 在终端中通过命令 npm install hexo -g安装
    • 博主看的教程没有专门说在哪个文件夹下安装，博主是在node_global文件夹下安装的，不知有无影响。
    • 直接使用install下载可能会速度较慢，建议开魔法
    • 可能会存在权限问题，不知道为什么我用管理员权限打开Git bash还是权限不够，所以后面是用管理员权限打开cmd，在cmd上操作的(但截图是在bash上截的，操作过程和信息提示都差不多，应该不影响？)。具体方法为在开始菜单搜索cmd，然后选择以管理员身份打开
    • 还有一个解决权限问题的方法是，右击node_cache和node_global文件夹，选择属性-安全，直接修改对应权限
  • 用 hexo -v 测试是否安装成功
    • 出现无法识别hexo指令错误
    • 解决方法：进入node_global文件夹查看，发现出现了一个node-jsnode_cache文件夹，里面有一个node_modules正是我所需要的，因此把node-jsnode_cache里的所有文件剪切到了node_global文件夹下，替换了之前自己新建的node_global/node_modules文件夹（反正它现在里面还是空的），然后把node-jsnode_cache删掉了，再尝试hexo -v，出现正确信息
    • 不知道这样做会不会有什么后续影响，如果后面发现了我再更新
    • 更新：不要在node_global文件夹里操作，可以退出来在上一层文件夹node-js里执行命令，就不会出现node-jsnode_cache影响后续操作
  • 用 npm install --save hexo-deployer-git 安装hexo依赖
    • 这里是我验证了退出来node-js文件夹操作还会不会出现node-jsnode_cache，所以多了后面的一步验证操作，如果有不放心的小伙伴也可以在后面再用hexo -v检查一次

Git配置SSH key

注册一个github账号

如果没有github，此处需要注册一个，具体的注册过程就不赘述啦，会看到这里的估计都有吧

配置SSH

  • 在终端使用 cd ~/.ssh 查看以前是否配置过SSH
    • 若出现 No such file or directory则说明以前没有配置过，接第二步
    • 否则说明以前配置过，直接进入第三步
  • 在终端输入ssh-keygen -t rsa -C "邮件地址" ，然后连续回车3次生成ssh key
    • 如ssh-keygen -t rsa -C "90695xxxx@qq.com"
  • 此时文件夹内会出现如图所示的.ssh文件夹和rsa文件，用记事本打开id_rsa.pub文件，复制里面的所有内容
  • 打开登录了账号的github主页，点击右上角头像，在出来的侧边栏里找到setting选项，点击进入设置页面，点击左侧的SSH and GPG keys，点击New SSH key
  • 将id_rsa.pub复制的内容粘贴到key中，title可以任意取，然后点击Add SSH key
  • 然后在git bash终端输入ssh -T git@github.com测试是否配置成功
    • 第一次测试需要根据文字提示输入一个yes，博主之前已经测试过了所以没有这一步
    • 成功之后会出现以下画面（我在cmd终端也输入了，但是在cmd会warn我permission are too open）
  • 再在终端输入以下两条指令配置账号密码，双引号内的用户名和邮箱改成自己的，
    • git config --global user.name "GiselleJy" ←Github用户名
    • git config --global user.email "90695xxxx@qq.com" ←Github注册时用的邮箱

搭建博客

  • 新建一个以后存放博客源文件的目录
    • 上面有提到，博主是在node-js的安装文件夹里新建了一个blog文件夹，但是感觉怪怪的。。。
  • 在git bash进入本地博客的存放目录，然后执行初始化命令 hexo init （这里不能用cmd…）
    执行完毕后可以在blog文件夹里看到一堆文件和文件夹
  • 使用hexo g 指令生成静态网页，然后使用hexo s指令进行试浏览
    • 注意，这里**没有-**，不是hexo -g
    • 因为我之前添加过butterfly模板，所以这里有butterfly的提示框。如果之前没有额外选择过模板，就是默认的landscape模板，不会有下面那个大大的butterfly字样
    • hexo s执行完后可以去http://localhost:4000/在本地初步浏览自己的页面

部署到github

  • 新建一个GitHub仓库
    • 注意！！ 仓库名一定是 github用户名.github.io
  • 编辑_config.yml文件
    • 在存放blog的那个文件夹下有个_config.yml文件，打开进行编辑，在末尾添加
    • 保存文件需要管理员权限，如果后续步骤受到影响可以返回来检查这个文件有没有成功保存
  • 用hexo d命令将本地的文件发布到github
  • 在github页面进行查看
    • 如果刚刚在设置_config.yml时和我一样忘记改branch，需要在这里将下面的branches设置到和config文件中的一致
