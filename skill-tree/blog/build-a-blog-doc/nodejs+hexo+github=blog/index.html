# 环境搭建

# git

  • 进入 git 官网下载安装包，进行安装
  • 安装好后打开 Git Bash，输入 git -v ，正确显示版本说明安装成功

# node.js

  • 进入 node.js 官网下载安装包进行安装
  • 安装好后的 nodejs 文件夹里可能只有 node_modules 文件夹，需要手动新建 node_global 和 node_cache 文件夹
  • 配置环境变量，详细配置过程可参考这一篇博客
  • 另：该博主还记录了一篇完全删除 node.js 的博客

# Hexo

  • 这部分参考了技能树写得非常漂亮的这位兄弟，全局安装完 hexo 后，创建一个博客专用文件夹，然后再以管理员身份打开 git bash，进入该文件夹后再执行 hexo init 对该文件夹进行初始化。

# 配置 Github 的 SSH key

  • 在用户根目录使用 cd ~/ssh 查看以前是否配置过 SSH
    • 若出现 'No such file or directory' 说明以前没配置过
    • 否则说明以前配置过，跳过下一步
  • 继续输入 ssh-keygen -t rsa -C "9069xxxxx@qq.com" 然后连续回车 3 次生成 ssh key, 此时～/ssh 文件夹内会出现如图所示的两个文件
  • 用记事本打开 id_rsa.pub 文件，将内容全部复制到 github 主页 → setting → SSH and GPG keys → New SSH key 中，title 随便取，然后点击 add SSH key
  • 可在 git bash 输入 ssh -T git@github.com 测试是否配置成功
    • 第一次测试时需要根据文字提示输入一个 yes

# 搭建博客

# hexo 常用指令

  • 至此终于可以开始编辑属于自己的博客啦！可以选择自己喜欢的模板开始装扮博客，有能力也可以自己编写主题

  • 常用指令如下：hexo 官方指令文档

    hexo clean & hexo g & hexo s
hexo clean & hexo g & hexo d
    功能指令效果
    新建 md 文件hexo n 文件名在 source 文件夹下创建一个 文件名.md 文件，再使用 md 文本编辑器进行编辑
    发布hexo clean清除缓存文件 (db.json) 和已生成的静态文件 (public)。
    /hexo g生成静态文件。
    /hexo s启动服务器。 默认情况下，访问网址为： http://localhost:4000/ 。
    /hexo d网站部署。

# 路径管理

  • 展示一下到此为止我的文件路径：

    blog 分支中的：
    1. _config.yml 文件：站点配置文件，很多功能、插件需要修改该配置文件应用生效。
    2. node_modules 文件夹：包含依赖的模块。
    3. package.json 文件：依赖的模块列表。说明见：package.json:Specifics of npm’s package.json handling
    4. package-lock.json 文件：依赖的模块安装记录。说明见：npm-package-locks:An explanation of npm lockfiles
    5. public 文件夹 ：包含生成的网页静态文件。
    6. scaffolds 文件夹 ：包含创建的文章、分类、标签界面的模板。博客的定制修改会对模板进行修改。
    7. source 文件夹 ：包含生成网页所需要的源文件，包括包含我们心血的 Markdown 文稿，这也是最重要的内容。
    8. themes 文件夹 ：其中 landscape 是默认的主题，其他文件夹是克隆下来时的主题。
    9. _config.shoka.yml 文件：针对 shoka 主题进行的配置修改文件。
